## Remix

Remix est un framework bâti sur React depuis 2022, par la même équipe que React Router. Financé par Shopify.

Autres frameworks bâtis sur React:

- Next JS
- Create-react-app (CRA)
- Gatsby
- T3
- Vite

---

### React vs Next JS

Next JS est plus populaire que Remix. (Surtout parce que Remix est très récent.)

En octobre 2023, la popularité de Remix n'est que 0.5% de la popularité de Next JS.

Article qui décrit les différences entre React et Remix:

https://prateeksurana.me/blog/nextjs-13-vs-remix-an-in-depth-case-study/

---

### Caractéristiques de Remix

- Framework complet
- Encourage l'usage de html/js standardisé: `Request`, `Response`, `FormData`, etc.
- Permet un layout imbriqué
- Chargement de données en parallèle
- Gestion des situations de concurrence (race conditions)
- Permet d'afficher le HTML avant le chargement du JS

## Déploiement Remix

{/* prettier-ignore */}
- Ne supporte pas encore de **_React Server Components_** (RSC)
  * RSC est prévu pour une version future
- Ne supporte pas de **_Static Side Generation_** (SSG)
  - On peut faire du caching. 
- Utilise une forme de **_Server Side Rendering_** (SSR) optimizé pour le dynamique
  - À cause du SSR, il faut déployer Remix sur Node JS (ou un serveur qui exécute du js)

### Démarrer avec Remix

```bash
npx create-remix@latest --install --init-script --git-init --template epicweb-dev/epic-stack
```

```bash
mkdir my-remix-app
cd my-remix-app

# install runtime dependencies
npm i @remix-run/node @remix-run/react @remix-run/serve isbot react react-dom

# install dev dependencies
npm i -D @remix-run/dev

```

### Routage Remix

Chaque fichier de notre route possède maintenant 3 sections:

- default (composant React)
- loader (charger data dans state)
- action (événements pour modifier data)

Les appels fetch dans useEffect ne sont plus nécessaires. On isole ces appels dans ces blocs `loader` et `action`.

Similaire à React-Router 6.4+, SAUF que loader et action roulent du côté serveur.
